ResearchSuite
ResearchSuite is an extension of the ResearchKit (iOS) and ResearchStack (Android) open source frameworks for developing research study apps, which allow for researchers to easily develop intuitive and standardized data-collecting mobile apps.

ResearchSuite extends the available surveys to include adaptable visual assessments and custom active tasks and supports integration of various backend modules.

On a high level, the structure of a ResearchSuite app is defined by a JSON file, which specifies the survey or active task steps to be instantiated by the app.

Limbr
Limbr is a compliance enhancement mHealth clinical trial that packages self-directed rehabilitation tutorial videos; personalizable visual self-report tools; health coach support; and sensor-assisted, passive activity-level tracking into a suite of smartphone-based apps for patients with CLBP.

The Limbr program aims to promote adherence to the Back Rx exercise rehabilitation regimen, increase engagement in self-directed management of pain (including pain, medication and exercise tracking), and improve self-reported outcomes for pain.

EmoTrain
EmoTrain is an interactive platform that is designed to target deficits in recognizing and labeling facial expressions and reciprocating emotionally, and to help teach these face processing skills to individuals with Autism Spectrum Disorder (ASD). EmoTrain is designed to teach individuals with ASD to recognize 7 basic human emotions (happy, sad, fear, disgust, anger, neutral, suprise) and to also perform those 7 facial expressions.

When playing EmoTrain, users are asked to attempt to match the images of face expressions they are presented with on the screen. The platform is able to track the user's expressions in real-time and judge if the user is performing the expressions correctly.

Dlib is used for real-time 64-point landmark tracking on a mobile device and OpenCV for face detection and client-side image processing. A CNN-based model is used for real-time expression recognition.

EmoTrain was a semi-finalist at the Standard Chartered Women + Tech4NYC Startup Competition at the Zahn Innovation Center in NYC.

Assist
ASSIST is a navigation application designed for visually impaired users, capable of tracking the position of the user within an indoor environment and providing turn-by-turn directions to a location (i.e. elevator, room 52, cubicle 7) using BLE Beacons, Tango, or hybrid infrastructure depending on infrastructure availability.

The hybrid Beacon-Tango localization system designed is capable of highly accurate, robust, real-time multi-floor indoor localization. Bluetooth Low Energy (BLE) Beacons, which are excellent at coarse location detection are used for approximating the user's general location (i.e. floor or section within a large indoor environment). A vision-based approach, utilizing a Google Tango RGBD + 6-degrees-of-freedom visual inertial odometry enabled device (phone & phablet available), allows us the software to detect the fine location (position and orientation) of the user. The Tango device allows us to effortlesly and quickly record a 3D feature map of the environement in a compressed format, called an Area Description File (ADF), which allows for real-time re-localization within that environment. Although the Tango device was prone to small errors during the estimation of each camera position, the average error was 3 cm and the maximum error recorded was 0.4 m. A limitation experienced was the size of the ADF able to be recorded, which was found to be approximately a floor of a building. A Tango device is therefore used to record an ADF for each floor (or each section in a large building) and BLE Beacons can be used to detect the coarse location and load the corresponding ADF to be localized against for that section of the building. The software designed also has modules for being used by those without access to a Google Tango device, in which case it will only use BLE Beacon based localization.

Furthermore, an adaptable interface is designed which allows the position of the user to be tracked on a 2D floor plan of the environment - accomplished by aligning the 3D Area Description File to the 2D floor plan. An annotation module is also included which allows for waypoints to be placed on the 2D map associated with the 3D ADF.





SmartCane
The SmartCane system is a robotic white cane and mobile device navigation software for visually impaired people. The system includes software for Google Tango devices that utilizes simultaneous localization and mapping (SLAM) to plan a path and guide a visually impaired user to waypoints within indoor environments. A control panel is mounted on the standard white cane that enables visually impaired users to communicate with the navigation software and is additionally used to provide navigation instructions via haptic feedback. Based on the motion-tracking and localization capabilities of the Google Tango, the SmartCane is able to generate a safe path to the destination waypoint indicated by the user.

EmoVIP
The Emo-VIP system is a real-time facial computing framework aimed at helping visually impaired people recognize the emotions of the people in close proximity to them. The framework consists of a

  1. Face Mobile App which allows a user to capture camera feed of human faces
  2. Web Service which handles all requests from the mobile app
  3. Emotion Computing Server
The Face Mobile App (available on iOS + Android), uses OpenCV to perform face detection in real-time. The emotion recognition model on the server is based on a CNN structure with an accuracy of 81%.

The interface is designed with a visually impaired user in mind: screen gesture recognition, audio and haptic feedback are used.

AVA PAM
AVA PAM is an Adaptable Visual Assessment based on the Photograph Affect Meter - a one-touch assessment of mood and emotional state. Developed by JP Pollak, Phil Adams, and Geri Gay from Cornell Tech and Cornell University, it has been rigorously validated as a measure of affect, intended for frequent or expeditious use where longer form measures such as PANAS or POMS are not feasible.

AVA PAM is built using ResearchSuite.
